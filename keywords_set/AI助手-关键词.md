## 3) 结合你当前站点配置的“落地级”Meta 优化要点（`rspress.config.ts`）
你现在的 `head` 多为站点级默认值（description/keywords/canonical 都固定）。对 SEO 更有效的调整方向是：

+ *_Canonical__*_：不要全站都指向首页。*应该做到“每个页面 canonical 指向自身 URL”，否则可能稀释收录与权重归因。
+ *_Title/Description 的页面级差异化__*_：核心落地页（如 `ai助手免费使用`）需要独立的 title/description，避免全站重复。
+ *_OG/Twitter Image__*_：确保存在且与页面主题相关（AI助手相关封面），提升分享点击率。

（是否能在 Rspress 对不同路由设置 head，取决于你是否用到页面 Frontmatter 或主题 API；至少要避免全站同一 canonical。）

## 4) 内容写法（更贴近搜索意图）
### 针对“免费/在线使用”类强意图词
页面必须回答清楚：

+ “免费”指什么：免费模型/免费额度/是否需要注册
+ “在线使用”怎么用：3 步（打开链接 → 登录 → 开始对话）
+ 限制条件：额度、功能差异（透明更利于转化与长尾收录）

### 针对“写作助手”
用“模板 + 示例”覆盖 Query：

+ 文章/邮件/周报/文案/改写/润色/降重/提纲
+ 加入可复制的 Prompt 示例（你现有 `提示词管理` 页面内容可复用并内链过去）

### 针对“生成 PPT”
写清楚输入输出：

+ 输入：主题、受众、时长、风格
+ 输出：PPT 大纲 + 每页要点 + 演讲稿
+ 提供 1~2 个完整示例（更容易获得长尾排名）

### 针对“翻译助手”
强调质量与场景：

+ 商务邮件/合同/产品文档/术语一致性
+ 给出“术语表/风格要求”的指令示例

## 6) 技术侧（收录与页面体验）
+ 确保 sitemap 覆盖所有新页面：你已启用 `rspress-plugin-sitemap`，新增页面后应自动进入站点地图（前提是路由可被生成）。
+ 避免图片影响：为关键图片补充 `alt`（尤其是功能示意图），并压缩体积。
+ URL 语义化：用英文短路径（如 `/ai-assistant/free/`），页面内用中文关键词。
+ 首屏可读内容不要全靠图片：让爬虫能直接抓到关键词与答案段落。
